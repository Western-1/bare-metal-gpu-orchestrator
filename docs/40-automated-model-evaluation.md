# Automated Model Evaluation (LLM-as-a-Judge)

**Component:** CI/CD & Ragas  
**Objective:** Continuously evaluate model output quality and prevent accuracy regressions  
**Integration:** MLflow & ArgoCD Pre-Sync Hooks  

---

## 1. The Quality Regression Risk

In software engineering, Unit Tests validate code functionality. In Machine Learning, evaluating the output of Generative AI is non-deterministic.

When a Data Scientist pushes a newly fine-tuned LoRA adapter (`16-multi-lora-architecture.md`) to the MLflow registry (`18-mlflow-registry.md`), confirming that it doesn't crash (OOM) is insufficient. The orchestrator must mathematically verify that the model's contextual accuracy, hallucination rate, and structural coherence have not degraded compared to the current production baseline.

---

## 2. LLM-as-a-Judge Architecture

Traditional NLP metrics (BLEU, ROUGE) correlate poorly with human judgment for complex generative tasks. The enterprise standard utilizes an **LLM-as-a-Judge**.

A massive, highly capable foundation model (e.g., GPT-4 or a local 70B parameter model) is prompted to explicitly evaluate the outputs generated by the smaller, fine-tuned candidate model (e.g., a 7B parameter adapter).

### Ragas Framework

We utilize **Ragas** (Retrieval Augmented Generation Assessment) to automate this evaluation within the CI/CD pipeline.

**Core Metrics Assessed:**
- **Faithfulness:** Does the answer derive strictly from the provided context (no hallucinations)?
- **Answer Relevance:** Does the answer directly address the prompt?
- **Context Precision:** Is the retrieved context highly relevant?

---

## 3. Evaluation Pipeline Implementation

This pipeline executes autonomously within a Kubernetes Job (triggered via ArgoCD Pre-Sync or GitHub Actions) before the new model is promoted to production.

```python
# evaluate_model.py
from datasets import Dataset
from ragas import evaluate
from ragas.metrics import faithfulness, answer_relevancy

def run_evaluation():
    # 1. Load Gold Standard Dataset (Questions & Ground Truth)
    dataset = load_evaluation_set()
    
    # 2. Generate Answers using the *Candidate* Model
    answers = []
    for q in dataset['question']:
        response = query_candidate_vllm_api(q)
        answers.append(response)
        
    dataset['answer'] = answers

    # 3. Evaluate using the *Judge* Model (e.g., Llama-3-70B-Instruct)
    # The Judge evaluates the Candidate's answers against the context
    result = evaluate(
        Dataset.from_dict(dataset),
        metrics=[faithfulness, answer_relevancy],
    )
    
    # 4. Enforce Quality Gate
    if result["faithfulness"] < 0.90:
        raise Exception(f"Quality Gate Failed: Faithfulness degraded to {result['faithfulness']}")
        
    print("Quality Gate Passed. Approving production rollout.")
```

---

## 4. Continuous Deployment Integration

1. **Trigger:** A new model is registered in MLflow as `Staging`.
2. **Evaluation Job:** A Kubernetes Job spins up, downloading the Staging model and processing the `evaluate_model.py` script.
3. **Telemetry:** The Ragas evaluation scores are logged directly back into MLflow as metric artifacts for the specific model run.
4. **Promotion:** If the script exits with `0` (Gate Passed), an automated API call updates the MLflow tag to `Production`.
5. **Rollout:** ArgoCD detects the tag mutation and triggers the live FastAPI/vLLM endpoints to execute a rolling restart, pulling the validated weights into VRAM.

This ensures strict, autonomous, and mathematically rigorous quality control over generative deployments.
