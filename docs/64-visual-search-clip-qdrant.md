# Visual Search (CLIP & Qdrant)

**Component:** Image-to-Image Search (Coutury Core)  
**Objective:** Perform sub-millisecond similarity searches across 10,000+ fashion items  
**Architecture:** OpenAI CLIP (ViT) + Qdrant (HNSW)  

---

## 1. The Fashion Search Problem

Traditional e-commerce relies on text-based SQL queries (e.g., `SELECT * FROM pants WHERE color='black'`). 

Coutury users do not search with text. They upload a photo of a specific streetwear jacket and expect the app to find visually identical or stylistically matching items from the catalog. Text keywords cannot accurately describe the exact cut, texture, or aesthetic vibe of a garment.

---

## 2. Multimodal Embeddings (CLIP)

To solve this, the architecture utilizes **CLIP** (Contrastive Language-Image Pre-Training) or modern alternatives like **SigLIP**.

CLIP maps both text and images into the exact same high-dimensional mathematical space (e.g., a 512-dimensional vector). 
- The vector for the image of a black leather jacket.
- The vector for the text string "a black leather jacket".

Both vectors will have a very high cosine similarity. By embedding the cleanly cropped user image (`63-background-removal-segmentation.md`), we obtain a mathematical representation of its aesthetic.

---

## 3. Qdrant Vector Database Schema

Qdrant (`34-rag-vector-database-qdrant.md`) is designed to search millions of these vectors using HNSW (Hierarchical Navigable Small World) graphs.

For Coutury, the Qdrant Collection must store not just the vector, but the relational metadata (Payload) to allow for hybrid filtering (e.g., "Find jackets that look like this image, BUT they must be in stock and under $100").

### 3.1 Collection Creation

```python
from qdrant_client import QdrantClient
from qdrant_client.models import Distance, VectorParams

client = QdrantClient(host="qdrant.data-plane.svc.cluster.local", port=6333)

client.create_collection(
    collection_name="coutury_catalog",
    vectors_config=VectorParams(
        size=512,  # Matches the dimension of the CLIP model
        distance=Distance.COSINE
    )
)

# Create indices on the payload for ultra-fast hybrid filtering
client.create_payload_index("coutury_catalog", "category", field_schema="keyword")
client.create_payload_index("coutury_catalog", "price", field_schema="float")
```

### 3.2 Indexing the Catalog

When a new item is added to the Coutury inventory, its image is embedded and stored.

```python
# Embedding generated by Triton/CLIP
item_vector = [0.012, -0.453, 0.129, ...]
```

## 4. The Inference Query (Visual Search)

When a user uploads a photo to find an outfit match, the backend orchestrates the search.

```python
async def find_similar_items(user_image_bytes, user_budget: float):
    # 1. Remove background (Doc 63)
    clean_image = await preprocess_outfit(user_image_bytes)
    
    # 2. Generate Vector via CLIP
    query_vector = embed_image_with_clip(clean_image)
    
    # 3. Hybrid Search in Qdrant (Visual Similarity + SQL-like Filters)
    search_results = client.search(
        collection_name="coutury_catalog",
        query_vector=query_vector,
        limit=5,
        query_filter=models.Filter(
            must=[
                models.FieldCondition(
                    key="category",
                    match=models.MatchValue(value="Jacket")
                ),
                models.FieldCondition(
                    key="price",
                    range=models.Range(lte=user_budget)
                )
            ]
        )
    )
    
    return [hit.payload["image_url"] for hit in search_results]
```

**Result:** The user receives the top 5 visually matching jackets that fit their specific budget, computed in less than 50 milliseconds.
